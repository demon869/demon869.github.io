1.反解域和正解域的区别

  • DNS正解：根据域名===>DNS服务返回IP
  • DNS反解：根据IP===>DNS服务返回域名

PTR：IP===>FQDN，只能定义在反向区域数据中，反向区域名称为逆向网络地址加 .in-addr.arpa. 后缀组成
示例：

注意

  • 1、一般而言，维护主机是需要正解+反解的
  • 2、维护业务域，只需要正解即可
  • 3、主机域可以是假的，并未购买的，业务域推荐用真实的已购买的

2.添加反解域的自定义区域配置

vim /etc/named.rfc1912.zones

3.添加反解域的区域数据库文件

vim /var/named/91.168.192.in-addr.arpa.zone

注意：一个IP只能对应唯一的FQDN反解PTR记录，且应该与正解A记录对应
